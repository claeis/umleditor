4.2 Dialogues

Il existe pour chaque élément de modélisation un dialogue spécifique permettant la visualisation et la modification des caractéristiques qui lui sont propres.

Chaque dialogue d’élément de modélisation présente au moins les volets suivants :


FIG. 4.6: (Volet Description) du dialogue

CHAMP

DESCRIPTION

Description

Saisie de texte avec un menu-fenêtre comportant les fonctions de texte courantes (cf. § 3.1.2). Ce texte sera pris en compte lors de la génération du catalogue d’objets.

Il est nécessaire, pour certains éléments de modélisation, de saisir la définition exacte sous forme de texte conformément aux règles propres au langage INTERLIS (cf. figure 4.7).


FIG. 4.7: (Volet Syntaxe) du dialogue

CHAMP

DESCRIPTION

Dépendant de

Les dépendances possibles envers d’autres éléments de modélisation peuvent être sélectionnées dans une liste de choix.

Syntaxe d’INTERLIS

Saisie de texte avec menu-fenêtre comportant les fonctions de texte courantes (cf. § 3.1.2). Il est possible de saisir un code de syntaxe (selon les règles propres à INTERLIS). Ce code est ensuite vérifié par la fonction Contrôler le modèle (cf. § 3.1.7).

Chaque dialogue intègre les fonctions suivantes :

4.2.10 Attribut

Un attribut (feature ou property en anglais) est un élément de données d’une classe auquel un nom et un type de données sont associés (cf. figure 4.30). Les types de données parfaitement définis par INTERLIS sont à la disposition de l’utilisateur.
Les attributs peuvent être gérés de trois manières différentes, à savoir dans la zone de navigation (cf. § 3.3), dans le diagramme de classe cf. § 4.1.2 ou dans ce dialogue.

Vous voudrez bien vous reporter au manuel de référence d’INTERLIS 2 - 2.6 pour de plus amples informations relatives aux règles et aux propriétés de ce langage.
